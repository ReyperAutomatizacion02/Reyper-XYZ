# Documentation Standards: Reyper XYZ

ALWAYS apply these rules when creating or updating documentation in Notion:

## 1. Hierarchy and Structure
- **Root Page Strategy**: DO NOT create sub-pages. Keep all documentation on the root page `Reyper XYZ`.
- **Table of Contents**: Maintain a dynamic Table of Contents at the top of the root page.
- **Hierarchy**: 
    - Title 1 (#) for Major Modules.
    - Title 2 (##) for Main Sections.
    - Title 3 (###) for Technical Details.
- **Dividers**: Use dividers (---) between major sections to maintain visual flow.

## 2. Visual Richness and Aesthetics
- **Colors**: Apply colors to text or backgrounds (e.g., Red for critical, Gray for context, Green for status).
- **Emojis**: Use relevant emojis at the start of EVERY title and liberally throughout the text.
- **Callouts (Destacados)**: Use for summaries, key notes, or warnings. Color code them appropriately.
- **Columns**: Use columns for parallel lists (like company areas) or comparing concepts.

## 3. Content Elements
- **Toggles (Desplegables)**: Mandatory for large technical blocks, code snippets, or lengthy descriptions to keep the UI clean.
- **Lists**: 
    - Bullet points for features/items.
    - Numbered lists for sequential processes.
    - Task lists (Checkboxes) for progress tracking.
- **Quotes (Citas)**: Use for philosophy, "why" statements, or important project quotes.
- **Code Blocks**: Always use technical code blocks for SQL, JS, or command-line instructions.

## 4. Depth of Detail
- Document "hasta el más mínimo detalle" (down to the smallest detail).
- Include the "Why": Always explain the rationale behind architectural or feature decisions.
